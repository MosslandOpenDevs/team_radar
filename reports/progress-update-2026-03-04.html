TeamRadar 작업 공유

작성일: 2026-03-04 (UTC) · 대상: 내부 업무공유
TeamRadar의 데이터 수집·시각화·성능 개선 작업을 한 번에 공유할 수 있도록 구성한 리포트입니다.
Discord 근태/업무 채널 수집 Composed Map 연동 2분 폴링 업데이트 렌더링 성능 최적화

1) 이번 작업 목표

  • Discord 수집 데이터(근태/업무)를 맵 시각화와 대시보드에 안정적으로 연결
  • 기존 맵(composed_set_map.html) UX 유지 상태에서 데이터 연동
  • 업무 이벤트 기반 캐릭터 구성, 근태 상태 표시, 최근 이벤트 반영
  • 운영 성능 개선(렌더링 최적화 + 주기 폴링)

2) 핵심 반영 사항

데이터/서버
  • dashboard-server.js: CORS 허용 추가 (8765 맵에서 3100 API 호출 가능)
  • discord-monitor.js: 업무 백필(최근 7일) 추가
  • 데이터 수집 범위는 Discord의 근태 채널, 업무공유 채널로 제한하여 운영
  • 환경변수: STARTUP_WORK_BACKFILL=true, WORK_BACKFILL_DAYS=7
맵 연동 (기존 composed 맵 유지)
  • map/composed_set_map.html에서 :3100/api/team/status, :3100/api/team/logs 폴링
  • 맵은 생성형으로 자동 생성한 것이 아니라, 배경/오브젝트 에셋을 조합(composed)해서 구성한 형태
  • 2분 주기 갱신 (DATA_POLL_MS=120000)
  • 사용자별 캐릭터 생성 + 근태/업무 상태 표시
  • 최근 업무 이벤트(최대 5개) 기반 말풍선 반영
  • 근태 alias 매핑(attendanceAlias → todayAttendanceByName)으로 정확도 보강
대시보드 개선
  • 근태 매핑 UI 유지
  • 사용자별 캐릭터 선택 UI/API 추가 (/api/team/characters/apply)

3) 성능 최적화

  • 뷰포트 컬링: 화면 영역(+margin)만 렌더링
  • 액터 컬링: 보이는 캐릭터만 정렬/드로우
  • 텍스트 측정 캐시: 닉네임/상태/말풍선 폭 캐싱
  • 정보 패널 갱신 스로틀: 매 프레임 → 1초 주기

5) 다음 개선 제안

  • 업무 이벤트 메시지 개선: 현재 요약 중심 표시를 확장해, 업무 이벤트를 구조화(키워드/진행상태/우선순위/담당/링크)하여 맵과 대시보드에서 더 정확하게 표현
  • 근태 상황별 맵 구성 분기: 근태 상태에 따라 맵 연출 자체를 다르게 구성해 상황 인지가 빠르게 되도록 개선
